# TP Manage Sync - Integrated Deployment Guide

## Overview
This guide explains how to deploy TP Manage Sync (X12-UI + TpDataSync backend) as a single integrated service.

## Architecture
- **Frontend**: Angular app (X12-UI) served as static files
- **Backend**: .NET API service that also serves the frontend
- **Single Port**: Everything runs on one port (default 3100)
- **Single Process**: One `dotnet run` command starts everything

## Prerequisites
- .NET SDK (version used by TpDataSync project)
- Node.js & npm (for Angular build)
- Both repositories cloned:
  - `C:\GitHub\X12-UI`
  - `C:\CoPilot\TpDataSync\TpManageSync`

## Build Process

### Step 1: Build Angular & Integrate with Backend
Run from the X12-UI directory:

```powershell
cd C:\GitHub\X12-UI
.\build-integrated.ps1 -Configuration production
```

This script will:
1. Clean the backend's wwwroot folder
2. Build Angular in production mode
3. Copy all built files to the backend's wwwroot folder
4. Verify index.html exists

**Output**: `C:\CoPilot\TpDataSync\TpManageSync\server-dotnet\TpManageSync.Api\wwwroot\`

### Step 2: Start the Integrated Service
After the build completes, run:

```powershell
cd C:\CoPilot\TpDataSync\TpManageSync\server-dotnet\TpManageSync.Api
dotnet run
```

Wait for the output:
```
Now listening on: http://localhost:3100
```

### Step 3: Access the App
Open your browser and navigate to:
```
http://localhost:3100
```

You should see the X12-UI login page, and from there you can access TP Manage Sync via the Utilities menu.

## Configuration

### Environment Variables
The .NET backend supports the following environment variables:

```powershell
# Set before running dotnet run:
$env:API_PORT = "3100"                      # Default: 3100
$env:IRIS_NAMESPACE = "EDIPAYER"           # Source namespace default
$env:IRIS_DEST_NAMESPACE = "MISC"           # Destination namespace default
$env:IRIS_CONNECT_TIMEOUT_MS = "15000"      # IRIS connection timeout
```

### API Base URL
The frontend automatically detects whether it's running in:
- **Development**: `http://localhost:4200` → uses `http://localhost:3100/api`
- **Production**: Any other port → uses relative path `/api` (same server)

This is handled by `ApiConfigService` - no manual configuration needed.

When ready to deploy to production or a test server:

```powershell
cd C:\GitHub\X12-UI
.\build-integrated.ps1 -Configuration production

cd C:\CoPilot\TpDataSync\TpManageSync\server-dotnet\TpManageSync.Api
dotnet run
```

Access app at `http://localhost:3100` (or your configured port)

## Deployment to Different Server

To deploy the integrated service to a different server:

1. **On Development Machine:**
   ```powershell
   cd C:\GitHub\X12-UI
   .\build-integrated.ps1 -Configuration production
   ```

2. **Copy to Target Server:**
   - Copy entire folder: `C:\CoPilot\TpDataSync\TpManageSync\server-dotnet\TpManageSync.Api\` to target server
   - Or publish as self-contained: `dotnet publish -c Release -r win-x64 --self-contained`

3. **On Target Server:**
   ```powershell
   cd C:\path\to\TpManageSync.Api
   dotnet run
   ```
   Or if self-contained:
   ```powershell
   cd C:\path\to\published\TpManageSync.Api
   .\TpManageSync.Api.exe
   ```

## Troubleshooting

### "Cannot reach API" Error
- Ensure backend is running on port 3100
- Check if port 3100 is in use: `netstat -ano | findstr :3100`
- Verify IRIS connection settings are correct

### "index.html not found" After Build
- Check that `build-integrated.ps1` completed without errors
- Verify files in `wwwroot\` folder exist
- Try building again: `.\build-integrated.ps1 -Configuration production`

### Angular App Shows 404
- This usually means `wwwroot\` is empty or build failed
- Check build script output for errors
- Verify Node.js and npm are installed: `node -v && npm -v`

### ngModel Errors for Disabled Credentials
- The connect page no longer has username/password inputs (now uses logged-in credentials)
- If you see old HTML, ensure frontend is fully rebuilt and not cached
